Columns and Sorting Tabs
You can specify columns and sorting options for the way you want the search results to display either before you run the search, or after you run the search. If you launch the search directly from the Criteria tab before setting the columns/sorting features, by default only the Subject Identifiers display on the Search Results tab. You can then come back to the Columns and Sorting tabs to expand the display options. You would then need to run the search again, having set the display parameters.
The selection you make on the Columns tab determines whether caIntegrator searches clinical or genomic data. It does so using the criteria you set on the Criteria tab, whether it is clinical, gene expression or imaging. The data that displays however, is based on the selection on the Columns tab. In other words, if you select clinical criteria on the Criteria tab, but select Genomic on the Columns tab, the data that displays on the Search Results tab is genomic data that is filtered by the clinical and other criteria you set on the Criteria tab.
1.
On the Columns tab, select the Clinical or Genomic radio button to search clinical data.
Clinical – Select the annotation elements that you want to display in the search results. All elements listed are column headers in the data uploaded to the study. You can make multiple selections on this list.
The Patient or Subject Identifier display by default.
Genomic – Select the Reporter Type :
•
Gene Name
•
Probe Set
2.
Use the Select All or All buttons to aid you in making your selections.
The column selection is saved as part of the query if you save it. See Saving a Query.
3.
Click on the Sorting tab and select the order of the Search Results, using conventional UI sorting controls.
Sorting parameters are saved as part of the query if you choose to save it using the Save Query feature. You can also sort: sort the results by clicking on a column name.
