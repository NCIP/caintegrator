1.    The Research Investigator indicates that he will define a new query.
2.    The system presents an interface to define the query specific to the type of entity in question. See the various associated Special Requirements.
3.    The Research Investigator defines the query criteria (default no criteria, returning all entries). The system allows the Research Investigator to add a single criterion at a time providing all the operators (=, >, <, wildcard match, etc.) appropriate to the field and the specified criterion value. By default all criteria are “anded” together. All clinical and imaging feature data fields may be used as criteria.
4.    The Research Investigator chooses the fields to be displayed in the result set.
5.    The Research Investigator defines the ordering of the result set.
6.    The Research Investigator executes the query.
7.    The Research Investigator saves the query, giving it a unique name and a description.
 

Appears in: Manage Queries: Basic Flow
